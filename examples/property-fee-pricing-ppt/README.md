# Single-skill example: property management fee PPT generation

This example shows how to use SenseNova-Skills' **PPT generation** ability with nothing but a single prompt — no input file at all — to have the agent build a ready-to-open PPTX given a topic, page count, color palette, style requirement, and audience.

> Storyline: given the topic / page count / palette / style / audience → automatically produce outline and asset plan → render per-page HTML → composite into a PPTX via screenshots.

[中文版 / Chinese](README_CN.md)

## Input

None. All content is generated by the agent based on the topic (property management fee pricing) and the style requirements in the prompt.

## Skills involved

| Step | Skill | Purpose |
|------|-------|---------|
| 1️⃣ Entry dispatch | [`sn-ppt-entry`](../../skills/sn-ppt-entry/SKILL.md) | Dispatch to the appropriate PPT generation mode based on user intent |
| 2️⃣ PPT generation | [`sn-ppt-standard`](../../skills/sn-ppt-standard/SKILL.md) | Standard mode: produce outline, assets, per-page HTML, then composite into PPTX |

## Walkthrough

Prompt the agent directly:

```text
Build a 26-page PPT about a property management fee pricing system. Use a black-and-white palette,
keep the style warm and gentle. Audience: property management staff and the property committee.
```

Routed via `sn-ppt-entry` into `sn-ppt-standard`, which sequentially produces:

1. The outline and asset plan (`outline.json` / `asset_plan.json` / `info_pack.json` / `style_spec.json` / `task_pack.json`)
2. 26 per-page HTML files (with images, icons, and styling assets)
3. The PPTX composited from the page screenshots

## Artifacts

- [`result/物业费定价体系_20260425_104934.pptx`](result/物业费定价体系_20260425_104934.pptx): ready-to-open PPTX (composited from per-page HTML screenshots)
- [`result/物业费定价体系_20260425_104934.zip`](result/物业费定价体系_20260425_104934.zip): zipped per-page HTML sources, handy for re-editing or browser preview

Inside the zip, the directory `物业费定价体系_20260425_104934/` contains:

- `pages/page_001.html ~ page_026.html`: HTML source for each page
- `assets/`: referenced styling and scripts (e.g. echarts.min.js)
- `images/`: referenced images (jpg)
- `outline.json` / `asset_plan.json` / `info_pack.json` / `style_spec.json` / `task_pack.json`: intermediate artifacts (outline, asset plan, style spec, etc.)
