# 图表类型选型决策树

## 快速选型表

| 数据形态 | 推荐 type | 触发条件 | 不推荐 |
|---------|----------|---------|--------|
| 同一指标随时间变化（月/季/年） | line | labels 是连续时间序列，数据集 ≤ 3 | bar（空间浪费，趋势不直观） |
| 多对象单指标对比大小 | bar | 对象 3-20 个，适合横向比较 | pie（>5 个类别可读性差） |
| 占比/构成分析 | doughnut | 类别 3-10 个，合计 ≈ 100% | radar（不适合比例展示） |
| 多维度综合评分 | radar | 维度 3-12 个，2-4 个对比对象 | bar（维度多时堆叠混乱） |
| 不同量级的指标混显 | mixed (bar+line) | 一组柱状 + 一组折线，量级差 > 5x | 单一 chart（副轴丢失） |
| 大量级单元排名 | horizontalBar | 对象 8-30 个，适合排名阅读 | 普通 bar（标签横排不可读） |
| 科目/预算累计对比 | stacked bar | 各科目分层，需要看总量 + 结构 | 分组 bar（总量对比不直观） |

## 决策流程

```text
数据来了：
  数据点 > 1000 个？
    → 降采样或换 ECharts，Chart.js 有性能上限
  数据点 < 3 个？
    → 用 KPI 卡片展示，不用图表
  
  labels 是时间序列？
    → 数据集 ≤ 3 条 → line
    → 有柱状对比 + 趋势线 → mixed (bar+line)
  
  需要看占比？
    → 类别 ≤ 10 → doughnut
    → 类别 > 10 → 合并次要项为"其他"→ doughnut
    → 有多个维度 → stacked bar
  
  需要比较大小？
    → 对象 3-8 → bar
    → 对象 8-30 → horizontalBar
    → 对象 > 30 → 表格展示，不用图表
  
  需要多维度评分？
    → 维度 3-12，对比对象 2-4 → radar
    → 维度 > 12 → 分组 bar + 图例
  
  都不匹配？
    → bar（最安全默认）
```

## 常见反模式

- 饼图展示时间趋势（×）→ 应该用 line
- 雷达图只展示一个数据集（×）→ 雷达图的价值在对比
- 折线图数据点 < 5 个（×）→ 用 bar 更直观
- 柱状图数据集过多（> 6 条 dataset）（×）→ 拆成多图或用 stacked bar
- 负值用 doughnut（×）→ 自动回退到 bar
